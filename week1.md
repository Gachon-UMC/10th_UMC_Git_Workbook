# 1주차 Git 정리

## 배운 내용

- git이란?
  정의 : 파일의 변화를 기록하고 찾아주며, 변경사항들을 추적해주는 분산버전관리 시스템이다.
  장점 :이와 같은 기능 덕분에 여러 사람들이 서로 충돌하지 않고 동시에 과제를 수행해 나갈 수 있다.

- 작업 공간
  1.working directory
  내가 현재 작업하는 공간이다. 기본적으로는 모든 파일의 변경 사항이 추적되지만 원격 레포지토리에 올리지 못하는 파일들은 .gitignore에 저장해 추적을 피하는 방법을 사용한다.
  2.staging area
  commit을 하기 위한 준비단계이다. 수정된 파일 중 선택적으로 커밋 할 수 있도록 도움을 준다. add를 통해 선택된 파일들만 staged상태로 만들어준다.
  3.Local repository
  staged 상태가 된 파일들의 변경사항을 기록하는 곳이다. commit을 통해 작동한다. 그리고 여기서 push를 통해 원격 레파지토리에 올리게 된다.

- git clone
  로컬 레파지토리와 원격 레파지토리를 연결해준다

- git branch
  개발자들끼리 서로 영향을 주지 않기 위해 기능별로 작업장을 나누는 것이다.

- git switch
  내가 원하는 브랜치로 이동할 수 있도록 해준다.

- git add
  특정 파일들을 staged 상태로 변경해준다.

- git commit
  staged된 파일들 중 원하는 파일들을 로컬 레파지토리에 올려준다.

- git push
  로컬 레파지토리에 있는 파일들 중 특정 파일들을 원격 레파지토리로 올려준다

- .gitignore
  워킹 디렉토리에서 추적되지 않아야 하는 파일 목록을 지정하는 파일이다.

- git status
  내 파일의 상태를 확인시켜준다

- git merge
  현재 브랜치를 다른 브랜치와 병합시키는 명령어이다. 여러 사람이 한꺼번에 작업하기에 충돌 위험이 있어 잘 사용하지 않는다.
- pull request
  작업한 브랜치를 메인 브랜치로 반영하기 위해 pull을 받아줄 것을 요청하는 것이다.
