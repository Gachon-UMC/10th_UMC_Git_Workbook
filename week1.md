# 1주차 Git & GitHub 학습 정리

## 1. Git이란?
- **정의:** 파일의 변경 사항을 추적하고 여러 사용자 간에 해당 파일들의 작업들을 조율하기 위한 **분산 버전 관리 시스템**입니다.
- **기능:** 파일 내에서 일어난 변화를 기록하고, 로그 형식으로 관리하여 특정 시점에서 이전 내용을 언제든지 다시 꺼내볼 수 있게 합니다.

## 2. 저장소(Repository)와 구성 요소
- **Repository:** 작업하는 파일들이 저장되는 폴더의 개념으로, 변경 사항이 추적되고 관리되는 공간입니다.
- **로컬 레포(Local Repo):** 사용자의 컴퓨터 내에 있는 저장소.
- **원격 레포(Remote Repo):** Github에 올라가 있는 저장소.
- **Working Directory:** 현재 작업하고 있는 실제 폴더 공간입니다.

## 3. 파일 상태 및 단계
- **Staging Area:** 커밋을 위한 준비 단계로, 수정한 파일 중 선택적으로 반영할 파일을 모아두는 곳입니다.
- **Modified:** 파일에 변동이 생긴 상태입니다.
- **Staged:** 커밋을 위해 준비된 상태를 의미하며, 이 과정을 **add**라고 부릅니다.
- **Committed:** 변경 사항을 체크포인트로 기록하는 것을 **commit**이라고 부릅니다.

## 4. 브랜치와 로그
- **Branch:** 개발자들이 동시에 작업하지만 서로에게 영향을 끼치지 않도록 처리해주는 독립적인 작업 영역입니다.
- **HEAD:** 현재 브랜치의 최신 커밋을 참조하는 포인터 값입니다.
- **git log:** 커밋 해시, 생성자, 일시, 메시지 등 그동안 남겼던 커밋 기록을 조회합니다.

## 5. 주요 명령어 정리
- `git clone (주소)`: 원격 레포와 로컬 레포 연결
- `git status`: 내 파일 상태 확인하기
- `git add .`: 모든 파일을 staged 상태로 변경
- `git commit -m "메시지"`: 로컬 레포에 변경 사항 기록
- `git push origin (브랜치)`: 원격 레포에 변경 사항 반영
- `git branch (이름)`: 브랜치 생성
- `git switch (이름)`: 브랜치 전환
