# Week1 Git 정리

## 1. Git이란?
Git은 파일의 변경 사항을 추적하고 여러 사람이 함께 작업할 수 있도록 도와주는 분산 버전 관리 시스탬이다.

파일을 수정할 때마다 버전을 저장해 이전 상태로 돌아갈 수 있고, 협업 시에도 동일한 파일을 공유하며 작업할 수 있다.

---

## 2. Git의 주요 기능

### (1) 변경 사항 추적
Git은 파일이 언제, 어떻게 변경되었는지를 기록한다.  
따라서 여러 버전의 파일을 따로 저장하지 않아도 이전 상태를 확인할 수 있다.

### (2) 협업 기능
Github와 같은 원격 저장소를 통해 여러 사용자가 같은 프로젝트를 공유하고 동시에 작업할 수 있다.

### (3) 분산 버전 관리 시스템
여러명이 전체 파일과 기록을 자신의 컴퓨터에 가지고 작업할 수 있는 구조이다.

---

## 3. Git 세부내용

### (1) Working Directory
현재 내가 작업하고 있는 공간이다.

### (2) Staging Area
커밋하기 전, 파일을 임시로 올려두는 공간이다.

### (3) Local Repository
commit을 통해 변경사항이 저장되는 공간이다.

---

## 4. 세부 내용

### add
변경된 파일을 staging area에 올리는 단계이다.

### commit
변경 사항을 저장하는 단계이다.

### push
로컬 저장소의 내용을 Github에 업로드하는 것이다.

---

## 5. .gitignore
특정 파일을 Git이 추적하지 않도록 설정하는 파일이다. 

예를 들어 보안 파일이나 필요 없는 파일을 제외할 때 사용한다.

---

## 6. Branch
브랜치는 작업을 독립적으로 진행할 수 있는 공간이다.

여러 사람이 동시에 작업해도 서로 영향을 주지 않고, 나중에 합칠 수 있다.

---

## 7. 느낀점
Git은 처음에는 어렵게 느껴졌지만, 파일 관리와 협업을 효율적으로 하는 것에 많은 도움을 주는 것 같다.
