## 1주차 배운 내용 정리

### git이란?

파일의 변경 사항을 추적하고 여러 사용자 간에 해당 파일들의 작업들을 조율하기 위한 분산 버전 관리 시스템



### gitignore

.gitignore에 a.txt을 입력하면 a.txt 파일이 추적되지 않게 관리할 수 있다.



### git add .

모든 파일을 staged 상태로 변경



### git commit -m "(커밋 메시지)"

내가 이번 커밋에서 어떤 작업을 했는지 간단하게 적어줄 수 있다.



### git push origin (브랜치 이름)

로컬 레포에 반영된 내용을 원격 레포를 보내기 위한 명령어

origin: 원격 레포지토리의 URL을 담고 있는 키워드



### Branch란?

한 시점에서 서로에게 영향을 주지 않는 작업을 진행하기 위한 것

보통 기능 단위로 많이 분리



git branch (브랜치 이름) -> 브랜치 생성

git switch (브랜치 이름) -> 새롭게 만든 브랜치로 이동

git merge (병합할 브랜치) -> 현재 브랜치에서 다른 브랜치의 변경사항을 현재 브랜치에 적용시키고 싶을 때 사용하는 명령어
