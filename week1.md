# 📚 1주차 Git & GitHub 정리

---

## 1. Git이란?

Git은 분산 버전 관리 시스템(Version Control System)으로,
파일의 변경 이력을 기록하고 관리할 수 있도록 도와주는 도구이다.

개발 과정에서 코드의 수정 내역을 추적할 수 있으며,
여러 사람이 동시에 작업할 때 충돌을 방지하고 협업을 원활하게 해준다.

---

## 2. Git의 특징

- 버전 관리: 코드의 변경 이력을 저장하고 이전 상태로 되돌릴 수 있다.
- 분산 시스템: 중앙 서버 없이도 로컬에서 모든 작업이 가능하다.
- 협업 용이성: 여러 개발자가 동시에 작업하고 병합할 수 있다.
- 브랜치 기능: 기능별로 작업을 분리하여 안정적인 개발이 가능하다.

---

## 3. Git과 GitHub의 차이

- Git: 로컬 환경에서 버전 관리를 수행하는 도구
- GitHub: Git 저장소를 원격으로 관리하고 협업할 수 있는 플랫폼

즉, Git은 도구이고 GitHub는 그 도구를 사용하는 서비스이다.

---

## 4. 이번 주에 배운 Git 명령어

### git clone

원격 저장소를 로컬로 복사한다.

### git checkout -b

새로운 브랜치를 생성하고 이동한다.

### git add

변경된 파일을 스테이징 영역에 추가한다.

### git commit

변경 사항을 저장한다.

### git push

로컬 변경 사항을 원격 저장소로 업로드한다.
