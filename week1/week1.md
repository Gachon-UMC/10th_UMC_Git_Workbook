# 1주차 스터디

## 1. TypeScript를 학습하는 이유

- 타입 검사로 런타임 오류를 줄여준다.
- 자동 완성과 코드 인텔리센스로 개발 생산성이 향상된다.
- 명확한 타입 덕분에 코드 이해가 쉽다.
- 기존 JS 코드에 TS를 점진적으로 적용할 수 있다.

---

## 2. TypeScript 기초 문법

- **변수 이름 바로 뒤에 콜론과 함께 타입을 표기한다.**
  `let name: string = yeon;`
- 함수의 매개변수와 반환값에 타입을 지정할 수 있다.
- 객체의 구조를 정의할 때 **인터페이스**를 많이 사용한다.
- **Union**은 |을 사용해 표현하며, 변수가 여러 타입을 가질 수 있도록 한다.
- **Generic**은 다양한 타입을 재사용하기 위해 사용한다. 구체적인 타입을 미리 정하지 않고, 나중에 사용하는 시점에 외부에서 타입을 지정할 수 있다.
- **Enum**은 이름이 있는 상수 집합을 정의하는 방법으로, 여러 값을 하나의 그룹으로 묶어서 가독성을 높이고, 의미 있는 이름을 제공해준다.
- **Utility** 타입은 기존 타입을 변형해 재사용성을 높이는 전역 타입 변환 도구로, `Partial`, `Pick` 등이 있다.
