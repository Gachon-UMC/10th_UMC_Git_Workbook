# 1주차 스터디: Git & GitHub 기초

## 1. Git이란?

- 파일의 변경 사항을 추적하고 여러 사용자 간의 작업을 조율하기 위한 분산 버전 관리 시스템이다.
- 번거로운 파일 복사 없이 하나의 파일 내에서 버전을 관리할 수 있다.
- 특정 시점의 기록(로그)을 언제든지 다시 확인하고 되돌릴 수 있다.
- 여러 개발자가 동시에 협업하며 작업 내용을 안전하게 합칠 수 있다.

---

## 2. Git의 주요 저장 공간

1. **Working Directory:** 실제 파일을 수정하고 있는 내 컴퓨터의 작업 폴더
2. **Staging Area:** 커밋을 하기 전, 반영할 파일들만 선택적으로 올리는 준비 영역 (`git add`)
3. **Local Repository:** 내 컴퓨터의 Git 디렉토리에 버전 기록이 최종 저장되는 곳 (`git commit`)
4. **Remote Repository:** GitHub과 같이 인터넷상에 존재하는 원격 저장소 (`git push`)

---

## 3. 핵심 기능

### 기초 명령어

- `git clone`: 원격 저장소의 전체 내용을 내 로컬 컴퓨터로 복제한다.
- `git status`: 현재 파일들의 수정 상태와 추적 여부를 확인한다.
- `git log`: 커밋 해시, 작성자, 날짜 등 그동안 쌓인 커밋 이력을 조회한다.

### .gitignore

- 보안상 중요한 설정값(API 키 등)이나 불필요한 빌드 결과물이 Git에 추적되지 않도록 설정하는 파일이다.
- 파일명이나 폴더명, 특정 확장자(`*.txt`) 등을 기입하여 관리 대상에서 제외한다.

---

## 4. Branch

- **Branch:** 메인 작업 흐름(main)에서 뻗어 나와 독립적으로 작업할 수 있는 가상의 작업 공간으로, 기능 단위로 생성하여 서로 영향을 주지 않고 개발할 수 있다.
- **HEAD:** 현재 작업 중인 브랜치의 가장 최신 커밋을 가리키는 포인터이다.
- **Merge:** 작업이 완료된 브랜치를 다른 브랜치와 합치는 과정이다.

### Issue, Pull Request (PR)

- **Issue:** 해결해야 할 버그, 구현해야 할 기능 등 프로젝트의 작업 단위를 등록하고 논의하는 창구이다.
- **Pull Request:** 내가 작업한 브랜치의 내용을 메인 저장소에 반영해달라고 요청하는 단계이다. 팀원의 코드 리뷰를 거쳐 최종적으로 병합된다.
